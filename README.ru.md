# ✍️ Author — Платформа для писателей на базе ИИ

> Писательская студия на базе ИИ для романистов, сценаристов и авторов историй.

**Author** — это инструмент для творчества с поддержкой ИИ, предназначенный для авторов художественной литературы. Он объединяет профессиональный текстовый редактор, интеллектуального ИИ-помощника и полноценную систему управления миром — в едином интегрированном приложении.

🌐 **Онлайн Демо**: [author-delta.vercel.app](https://author-delta.vercel.app)

📦 **Зеркало Gitee (Для Китая)**: [gitee.com/yuanshijilong/author](https://gitee.com/yuanshijilong/author)

## 💬 Почему я создал это

Я довольно давно использую ИИ — с ранних дней ChatGPT 3.5, до Gemini 2.0 Exp Thinking, и в итоге остановился на Gemini 2.5 Pro Thinking после эры ChatGPT o1.

Как романисту, мне глубоко важна способность ИИ работать с языком. Романы длинные, поэтому мне нужны модели с большим окном контекста и высоким показателем отзыва (recall). Но что по-настоящему тронуло меня в Gemini — это его персонажи. Бывали моменты, когда слова на экране вызывали у меня желание плакать. Это эмоциональный резонанс. Мне нужен текст, который охватывает всю сложность человеческого существования.

Затем возобладала тенденция оптимизации под программирование. Каждая компания начала оптимизировать ИИ для кода. Я думал, что это хорошо — пока Gemini 3.1 Pro не начал описывать своих персонажей в биологических и психологических терминах. Оптимизированные для кода модели начали деконструировать людей на биологические компоненты. Claude Opus 4.6 был еще хуже: каждый персонаж говорил с пиковой эффективностью — кратко, экономно, не как человек, а как машина в человеческой маске.

**Я больше не видел, чтобы модели понимали человеческую сложность. Им стало все равно, что люди *делают* — только то, чем люди *являются*. Они перестали показывать личность через поведение и эмоции, вместо этого навешивая на людей простые определения.**

Я наблюдал, как универсальность этих моделей уничтожается. Я не хочу, чтобы мы жили в холодном мире кода. Я создал этот проект, чтобы ИИ мог сохранить **наш собственный язык** — за пределами механических операторов.

> Всем авторам, сценаристам, энтузиастам, читателям и игрокам, которые используют этот проект: Я надеюсь, что вы сможете проявить лучшее в своем ремесле, создавать работы с человеческим теплом и поддерживать пламя нашего языка живым. 🔥

## ✨ Основные возможности

### 📝 Профессиональный редактор
- Текстовый редактор на базе **Tiptap** — жирный шрифт, курсив, заголовки, списки, блоки кода и многое другое.
- **Постраничная разбивка в стиле Word** с визуальным макетом (WYSIWYG).
- Поддержка математических формул **KaTeX**.
- Настраиваемые шрифты, размер шрифта, межстрочный интервал и цвета.
- Подсчет слов / символов / абзацев в реальном времени.

### 🤖 ИИ-помощник писателя
- **Мультимодельная поддержка**: ZhipuAI GLM-4 / DeepSeek / OpenAI / Google Gemini / Claude / SiliconFlow / Volcengine / Moonshot + пользовательские эндпоинты.
- **Умная загрузка моделей** — получение списка моделей из API в один клик с поддержкой различных форматов прокси (`/models`, `/v1/models`), с защитой от таймаута.
- **Продолжить / Переписать / Улучшить / Расширить** — генерация в один клик.
- **Иммерсивный движок (Ghost Text)** потоковый предпросмотр — просматривайте вывод ИИ в реальном времени, как в Cursor, с возможностью принятия/отклонения.
- **Свободный режим чата** — обсуждение сюжета, персонажей и настроек с ИИ.
- **Глобальная память ИИ (Context Engine)** — ИИ автоматически читает профили персонажей, развитие мира и предыдущие главы для сохранения логики сюжета.

### 📚 Управление миром и сюжетом
- **Древовидная структура** управления для персонажей, локаций, предметов, планов и правил написания.
- Три режима письма: **Веб-роман** / **Литературная проза** / **Сценарий**, каждый со своими выделенными полями.
- Категоризация с цветовой кодировкой и дизайном glassmorphism.
- Настройки автоматически внедряются в контекст ИИ.

### 💾 Управление данными
- **Local-first** — все данные хранятся в IndexedDB браузера и никогда не загружаются на серверы.
- **Система снапшотов** — ручная/автоматическая система сохранения версий с откатом в один клик.
- **Импорт/Экспорт проектов** — полное резервное копирование проекта в формате JSON.
- **Многоформатный экспорт** — экспорт текущей главы в один клик или пакетный экспорт (TXT / Markdown / DOCX / EPUB / PDF).

### 🌐 Интернационализация
- 🇨🇳 简体中文 / 🇺🇸 English / 🇷🇺 Русский / 🇵🇸 الفلسطينية (العربية)

### 🎨 Пользовательский опыт
- Теплые тона для защиты глаз / переключатель темного режима.
- Интерактивный тур для новых пользователей.
- Панель справки с горячими клавишами.

## 🚀 Быстрый старт

> 💡 **Настоятельно рекомендуется**: Большинству пользователей, которым нужно только ежедневное письмо и многоплатформенная облачная синхронизация, пожалуйста, [напрямую скачайте и установите клиент](https://github.com/YuanShiJiLoong/author/releases/latest). Развертывание исходного кода или Vercel рекомендуется только для продвинутых пользователей, которым требуется **вторичная разработка** или которые готовы самостоятельно настроить базу данных Firebase.

### Требования
- **Node.js** 18+
- **npm** 9+ или **pnpm** 8+

### Установка

```bash
# Клонируйте репозиторий
git clone https://github.com/YuanShiJiLoong/author.git
# Или используйте зеркало Gitee (в Китае быстрее)
# git clone https://gitee.com/yuanshijilong/author.git
cd author

# Установите зависимости
npm install
# Или используйте pnpm
# pnpm install
# pnpm approve-builds    # Требуется для pnpm для активации нативных пакетов

# Настройте переменные окружения (опционально)
cp .env.example .env.local
# Отредактируйте .env.local, добавив свои API ключи
# Вы также можете настроить их в панели настроек приложения
```

### Сервер разработки

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000), чтобы начать писать.

### Сборка для продакшена

```bash
npm run build
npm start
```

### ☁️ Настройка облачной синхронизации (Self-Deploy)

> 💡 **Совет:** Десктопный клиент (Windows/macOS) **имеет встроенный официальный сервер облачной синхронизации**, не требующий дополнительных настроек. Если настройка Firebase кажется утомительной, **настоятельно рекомендуется напрямую загрузить и использовать клиент**.

Если вы настаиваете на развертывании через исходный код или Vercel и хотите включить синхронизацию между устройствами, выполните следующие действия:

#### 1. Создайте проект Firebase

1. Перейдите в [Firebase Console](https://console.firebase.google.com/) → **Create Project**
2. Включите **Authentication** → Метод входа → **Google**
3. Создайте **Firestore Database** (Начать в production mode)
4. Установите правила безопасности (Security Rules) Firestore для ограничения доступа для каждого пользователя:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

#### 2. Настройте переменные окружения

Скопируйте `.env.example` в `.env.local` и заполните раздел Firebase:

```bash
NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id
```

> Вы можете найти эти значения в Консоли Firebase → Настройки проекта → Общие → Ваши приложения → Конфигурация SDK.

#### 3. Для развертывания Vercel

Добавьте те же переменные в **Vercel Dashboard → Настройки проекта → Переменные окружения**, затем выполните повторное развертывание.

> 💡 Ключи API Firebase спроектированы так, чтобы быть публичными (идентификаторы на стороне клиента). Безопасность данных обеспечивается аутентификацией Firebase + правилами безопасности Firestore, а не скрытием ключа API.

## 🔄 Обновление

### Пользователи десктопного клиента

Загрузите последнюю версию установщика со страницы [Releases](https://github.com/YuanShiJiLoong/author/releases/latest) и установите поверх вашей текущей версии. Ваши данные хранятся в профиле браузера/Electron и не будут потеряны.

### Обычные пользователи (Исходный код)

#### Вариант 1: Автообновление в приложении

Откройте **Меню Справка → О программе → Проверить наличие обновлений** и нажмите "Обновить сейчас". Это автоматически запустит `git pull → npm install → npm run build`.

> ⚠️ **Вы должны перезагрузить сервер после обновления, чтобы изменения вступили в силу.** В приложении отобразятся инструкции по перезапуску.

#### Вариант 2: Обновление вручную

```bash
# 1. Загрузите последний код
git pull origin main

# 2. Установите зависимости (если появились новые)
npm install
# или: pnpm install && pnpm approve-builds

# 3. Пересоберите (требуется для режима production)
npm run build

# 4. Перезапустите сервер
# Режим разработки: Ctrl+C для остановки, затем перезапуск
npm run dev

# Режим Production: Ctrl+C для остановки, затем перезапуск
npm start

# При использовании PM2:
pm2 restart author
```

> ⚠️ **Запуск `git pull` без перезагрузки сервера НЕ применит обновление.** Запущенный процесс Node.js по-прежнему использует старый код.

### Пользователи Vercel

Если вы развернули приложение через форк Vercel, просто синхронизируйте свой форк с upstream на GitHub — Vercel автоматически выполнит переразвертывание.

## 🔍 Конфигурация веб-поиска

Author поддерживает веб-поиск на базе ИИ для получения информации в реальном времени:

| Провайдер | Метод поиска | Дополнительная настройка |
|----------|--------------|-------------|
| Gemini (Native) | Встроенный Google Поиск | Дополнительная настройка не нужна |
| OpenAI / OpenAI Responses | Встроенный веб-поиск | Дополнительная настройка не нужна |
| DeepSeek / ZhipuAI / SiliconFlow / прочие | Внешний API поиска | **Требуется ключ поисковой системы** |

Для провайдеров без встроенного поиска, выберите поисковую систему и введите свой API Ключ:

### Tavily (Рекомендуемый — самый простой)

1. Посетите [tavily.com](https://tavily.com) и создайте аккаунт
2. Ключ API находится в Панели управления (формат: `tvly-...`)
3. В Авторских настройках (Author Settings) → Веб-поиск → Выберите **Tavily** → Вставьте ключ

> Бесплатный тариф: **1,000 запросов/месяц**

### Exa (Семантический поиск)

1. Посетите [exa.ai](https://exa.ai) и создайте аккаунт
2. Получите ваш ключ API с [Панели управления](https://dashboard.exa.ai/api-keys)
3. В Авторских настройках (Author Settings) → Веб-поиск → Выберите **Exa** → Вставьте ключ

> Бесплатный тариф: **1,000 запросов/месяц** — семантический поиск, оптимизированный для ИИ.

### Custom Search API URL (Пул Прокси)

Если вы настроили пул прокси Tavily/Exa, используя несколько бесплатных аккаунтов, вы можете настроить **пользовательский URL API**:

1. В Авторских настройках → Веб-поиск → Раздел настройки поисковой системы
2. Найдите поле "🔗 Пользовательский URL API (необязательно)"
3. Введите URL прокси, например `https://your-proxy.com`
4. Оставьте пустым, чтобы использовать официальный адрес по умолчанию

> 💡 Инструмент автоматически добавляет суффикс `/search` к вашему URL — не нужно добавлять его вручную.

## 🧠 Векторизация (Embedding) и RAG Поиск настроек

> 💡 Эта функция предназначена для крупных произведений со множеством настроек системы/мира (>20). Использование в стандартных небольших рассказах обычно не требуется.

### Что такое Векторизация?

По умолчанию, сессии ИИ загружают **все записи о настройках** в контекст. Для крупных миров это быстро исчерпывает лимит длины окна у ИИ.

**Векторный поиск (RAG)** работает по-другому: каждая запись настроек преобразуется в математический вектор. Во время разговора система автоматически рассчитывает семантическое сходство и внедряет только **наиболее актуальные элементы** в контекст — вместо того, чтобы отправлять абсолютно всё.

### Когда это необходимо?

| Сценарий | Рекомендация |
|----------|---------------|
| Установок/записей < 20 | Не требуется — полное внедрение работает идеально |
| Установок 20–100 | Рекомендуется — улучшает точность отзыва ИИ |
| Установок > 100 | Крайне рекомендуется — критические детали лора не будут забыты |

### Как настроить

1. Откройте **Настройки** → **Конфигурация API**
2. Найдите и включите переключатель "Отдельный API Embedding (Separate Embedding API)"
3. Заполните:

| Настройка | Описание |
|---------|-------------|
| **Embedding API Ключ** | Ключ API для модели embedding (можно использовать тот же ключ) |
| **Embedding Base URL** | Эндпоинт API (например, `https://api.openai.com/v1`) |
| **Embedding Модель** | Имя модели (см. рекомендации ниже) |

#### Рекомендуемые Модели

| Провайдер | Имя Модели | Заметки |
|----------|-----------|-------|
| OpenAI | `text-embedding-3-small` | Экономично, 1536 измерений |
| OpenAI | `text-embedding-3-large` | Высшая точность, 3072 измерений |
| ZhipuAI | `embedding-3` | Оптимизирована под китайский, 2048 измерений |
| SiliconFlow | `BAAI/bge-m3` | Многоязычная модель, доступен бесплатный тариф |

### Автоматическая Векторизация

- **Умный триггер**: Срабатывает через 3 секунды после того, как вы закончите редактировать лор.
- **Инкрементное обновление**: Обновляются только измененные записи, экономя время и квоту API.
- **Локальное хранилище**: Векторные данные безопасно хранятся в вашем браузере (IndexedDB).
- **Авто-инициализация**: При импорте проекта из JSON, система автоматически построит индексы в фоновом режиме.

### Ручное пересоздание
Вы всегда можете принудительно перестроить индексы в **Настройки → Конфигурация API → "Rebuild Vector Index"**.

### Рабочий процесс

```
Пользователь редактирует запись лора → 3с задержка → Вызов Embedding API для векторизации
```

## 💾 Документация по формату импорта настроек

Author поддерживает импорт настроек из множества форматов: **JSON / Markdown / TXT / DOCX / PDF**.

### Структура: Четыре уровня

Система импорта использует **структурные маркеры** для строгого разграничения следующих четырех уровней. Независимо от используемого формата, эта вложенная структура должна соблюдаться:
`Категория → Элемент → Поле → Значение (может быть многострочным)`

Поэтому, **любая пользовательская категория, пользовательский тег или пользовательское поле будут корректно распознаны** и восстановлены без потерь.

### Маркеры Форматов Структуры

| Рейтинг | Формат | Категория | Элемент | Поле | Многострочность |
|------|--------|----------|-------|-------|-----------|
| ⭐⭐⭐⭐⭐ (Без потерь) | **JSON** | `category` | `name` | `content` key-val | `\n` перенос строки |
| ⭐⭐⭐⭐⭐ (Без потерь) | **MD** | `## Имя категории` | `### Имя элемента` | `**Тег**: Значение` | Текст с отступом |
| ⭐⭐⭐⭐ (Без потерь) | **TXT** | `│ Имя категории` | `【Имя элемента】` | `〈Тег〉: Значение` | Текст с отступом |
| ⭐⭐⭐ (Высокая точность) | **DOCX** | Заголовок 1 | Заголовок 2 | **Жирный** Тег + двоеточие | Абзац с отступом |
| ⭐⭐ (Высокая точность) | **PDF** | `■ Имя категории` | `◆ Имя элемента` | `▸ Тег: Значение` | Текст с отступом |

### Шаблон формата Markdown (Рекомендуется)

Используйте `##` для категорий, `###` для названий элементов, и `**Тег**: Содержимое` для полей:

```markdown
## Любая Пользовательская Категория

### Иван Иванов

**Роль**: Протагонист
**Предыстория**: Родился высоко в горах.
  С детства изучал боевые искусства с дедом.
  Его мастер сказал: "Ты должен сам найти ответы."
**Особая Родословная**: Кровь полудракона

### Анна Белая
**Позиция**: Поддержка команды
**Характер**: Жизнерадостная, обращает внимание на мелочи
```

### Шаблон формата TXT

Используйте `│` для блоков категорий, `【】` для названий элементов, и `〈Тег〉:` для полей:

```
┌──────────────────────────
│ Любая Пользовательская Категория
└──────────────────────────

【Иван Иванов】
〈Пол〉: Мужской
〈Предыстория〉: Родился высоко в горах.
  С детства изучал боевые искусства с дедом.
  Его мастер сказал: "Ты должен сам найти ответы."
〈Особая Родословная〉: Кровь полудракона
```

---

### Шаблон формата DOCX

В Microsoft Word используйте **Стили заголовков и полужирный шрифт** для разметки структуры:
- **Заголовок 1 (Heading 1 / H1)** → Название категории (например, "Любая Пользовательская Категория")
- **Заголовок 2 (Heading 2 / H2)** → Имя элемента (например, "Иван Иванов")
- **Первый абзац текста** → **Полужирное Название Поля**: Содержимое (например, **Особая Родословная**: Кровь полудракона)
- **Продолжение текста** → Многострочное содержимое с отступом первой строки

### 💡 Проектирование совместимости и Примечания

1. **Автоматический откат тегов (Обратная совместимость)** — Если в TXT/Markdown отсутствует структурный маркер (например, используется `Имя: Иван` вместо `**Имя**: Иван`), система попытается автоматически распарсить его совместимым образом, *только если* `Имя` принадлежит к базовому набору известных полей. Однако, чтобы гарантировать 100% распознавание и извлечение *пользовательских* полей, вы должны добавить структурное форматирование.
2. **Обработка многострочных значений** — Всякий раз, когда вы начинаете с `**Тег**: ` или `〈Тег〉: `, любые последующие **строки с отступом в 2 пробела** будут рассматриваться как тело этого поля (извлечение многострочного значения) до тех пор, пока не встретится следующий структурный тег.
3. **Ограничения парсинга DOCX / PDF** — PDF-файлы извлекаются с использованием специально разработанных идентификаторов `▸`. Если импортируемый файл PDF/DOCX был сгенерирован не этой системой, эвристический парсер попытается извлечь содержимое наилучшим образом, но 100% точность не может быть гарантирована.
4. **JSON является наиболее полным** — JSON, безусловно, является самым строгим форматом и единственным, способным на 100% мигрировать все атрибуты (включая режим письма и информацию о проекте). Если вы просто меняете устройства для работы, настоятельно рекомендуется импорт/экспорт через JSON.

## 🔒 Конфиденциальность и безопасность данных

### Локальное хранилище (Безопасно)
- Все творческие данные (черновики глав, настройки, снапшоты) **хранятся 100% локально в вашем браузере (IndexedDB)** — никогда не загружаются ни на какой сервер!
- API Ключи хранятся в `localStorage` браузера.

### ⚠️ Поток данных при использовании свойств ИИ

При использовании ИИ (продолжение, чат и т.д.) следующие данные проходят через **сервер развертывателя**:
- Ваш **API Ключ**
- Исходный **текст**, который вы отправляете ИИ

```
Ваш Браузер → Сервер Развертывателя → ИИ Провайдер (Gemini/DeepSeek/др.)
```

**Если вы используете чужой развернутый публичный экземпляр**, несмотря на то, что тот кто развертывал ресурс пообещал не инспектировать логи, техническая возможность перехватывать данные существует. Поэтому:

1. ✅ Используйте общедоступную версию **только для быстрой проверки/тестирования**.
2. ⚠️ После того, как вы попробуете платформу, **немедленно удалите свой API Ключ** на сайте поставщика.
3. 🔐 **Для реальной работы — сделайте Fork и разверните собственный приватный экземпляр**, таким образом все данные проходят только через ваш собственный сервер.

> 💡 Развернуть собственный экземпляр очень просто: Форкните этот репозиторий → Развертывание в один клик на Vercel → Готово. Это занимает менее 5 минут.
