# ✍️ Author — منصة الكتابة الإبداعية المدعومة بالذكاء الاصطناعي

> استوديو كتابة مدعوم بالذكاء الاصطناعي للروائيين وكتاب السيناريو ورواة القصص.

**Author** هي أداة للكتابة الإبداعية بمساعدة الذكاء الاصطناعي مصممة لكتاب الخيال. تجمع بين محرر نصوص احترافي، ومساعد كتابة ذكي، ونظام كامل لإدارة بناء العوالم والقصص — كل ذلك في تجربة واحدة سلسة.

🌐 **العرض التجريبي الحي**: [author-delta.vercel.app](https://author-delta.vercel.app)

📦 **نسخة محاكاة Gitee (للمستخدمين في الصين)**: [gitee.com/yuanshijilong/author](https://gitee.com/yuanshijilong/author)

## 💬 لماذا قمت ببناء هذا المشروع

لقد كنت أستخدم الذكاء الاصطناعي منذ فترة — بدءًا من الأيام الأولى لـ ChatGPT 3.5، وصولاً إلى Gemini 2.0 Exp Thinking، واستقر بي الحال أخيرًا على Gemini 2.5 Pro Thinking بعد حقبة ChatGPT o1.

بصفتي روائيًا، أهتم بشدة بقدرة الذكاء الاصطناعي على التعامل مع اللغة. الروايات طويلة، لذا أحتاج إلى نماذج ذات نوافذ سياق قوية واستدعاء عالي (Recall). ولكن ما أثر فيّ حقًا بشأن Gemini هو شخصياته — كانت هناك لحظات جعلتني فيها الكلمات على الشاشة أرغب في البكاء. هذا هو الرنين العاطفي. أحتاج إلى كتابة تحتضن التعقيد الكامل لكونك إنسانًا.

ثم سادت الاتجاهات التي تركز على البرمجة. بدأت كل شركة في تحسين الذكاء الاصطناعي للتعامل مع الأكواد البرمجية. اعتقدت أنه كان شيئًا جيدًا — حتى بدأ Gemini 3.1 Pro في وصف شخصياته بمصطلحات بيولوجية ونفسية فقط. لقد بدأت النماذج المُحسّنة للأكواد في تجريد البشر وتحويلهم إلى مكونات بيولوجية. كان Claude Opus 4.6 أسوأ من ذلك: فقد كانت كل شخصية تحكي و تتحدث بأقصى درجات الكفاءة — بشكل موجز واقتصادي، ليس كإنسان، بل كآلة ترتدي قناعًا بشريًا.

**لم أعد أرى النماذج تفهم التعقيد البشري. لم يهتموا بما *يفعله* البشر — بل فقط بما *هم* عليه. لقد توقفوا عن إظهار الشخصية من خلال السلوك والعاطفة، وبدلاً من ذلك وضعوا تعاريف بسيطة على البشر.**

لقد شاهدت براعة هذه النماذج وهي تتبدد وتتدمر. لا أريد أن نعيش في عالم بارد من الأكواد. لقد قمت ببناء هذا المشروع حتى يتمكن الذكاء الاصطناعي من الحفاظ على **لغتنا الخاصة** — بعيدًا عن المشغلات الميكانيكية.

> إلى جميع المؤلفين وكتاب السيناريو والهواة والقراء واللاعبين الذين يستخدمون هذا المشروع: أتمنى أن تبرزوا أفضل ما في حرفتكم، وتنشئوا أعمالاً بلمسة إنسانية، وتحافظوا على شعلة لغتنا حية. 🔥

## ✨ الميزات الأساسية

### 📝 محرر احترافي
- محرر نصوص غني مدعوم بـ **Tiptap** — عريض، مائل، عناوين، قوائم، كتل برمجية، والمزيد
- **نظام ترقيم الصفحات بنمط Word** مع تخطيط مرئي (WYSIWYG).
- دعم المعادلات الرياضية **KaTeX**.
- خطوط وألوان وأحجام نصوص ومسافات أسطر قابلة للتخصيص.
- إحصائيات للكلمات / الحروف / الفقرات في الوقت الفعلي.

### 🤖 مساعد الكتابة الذكي
- **دعم مزودين متعددين**: ZhipuAI GLM-4 / DeepSeek / OpenAI / Google Gemini / Claude / SiliconFlow / Volcengine / Moonshot ونقاط نهاية (endpoints) مخصصة.
- **جلب النماذج بذكاء** — بنقرة واحدة قم باستدعاء القائمة الكاملة من الـ API، متوافق مع كافة صيغ الوكيل (`/models`, `/v1/models`) مع حماية من المهلة (Timeout).
- **إكمال / إعادة كتابة / تحسين / توسيع** — إنشاء المهام بنقرة واحدة.
- **محرك الكتابة الغامر (Ghost Text)** — معاينة تدفق مخرجات الذكاء الاصطناعي في الوقت الفعلي كما الـ Cursor مع خيار القبول/الرفض.
- **وضع الدردشة الحر** — ناقش أحداث القصة والشخصيات والإعدادات مباشرة مع الذكاء الاصطناعي.
- **الذاكرة العالمية للذكاء الاصطناعي (Context Engine)** — يقوم الذكاء الاصطناعي تلقائيًا بقراءة ملفات تعريف الشخصيات للفهم المستمر لبناء العالم والفصول، والحفاظ على اتساق القصة.

### 📚 التخزين السحابي ومدير إعدادات العوالم
- **إدارة شجرية متفرعة** لمعلومات الشخصيات والمواقع والعناصر والخطوط العريضة وقواعد الكتابة.
- ثلاثة أوضاع للكتابة: **رواية ويب (Web Novel)** / **أدب كلاسيكي (Literary Fiction)** / **سيناريو (Screenplay)**، ولكل منها حقول مخصصة.
- فئات مرمزة لونياً بتصميم زجاجي (Glassmorphism).
- يُتم إدراج الإعدادات ومعلومات الشخصيات الخاصة بك تلقائياً داخل سياق الذكاء الاصطناعي (AI Context).

### 💾 إدارة البيانات
- **الأولوية للمحلي** — يتم تخزين بياناتك محلياً بالكامل في متصفحك (IndexedDB)، ولا ترفع للخوادم أبداً.
- **نظام الحفظ اللحظي (Snapshots)** — أخذ لقطات للعودة للتغييرات بضغطة واحدة، بطريقة يدوية أو آلية.
- **استيراد وتصدير المشروع** — دعم الاستيراد بصيغة النسخ الاحتياطي JSON.
- **التصدير متعدد الصيغ** — دعم التصدير للفصل الحالي أو التصدير المجمع كملفات (TXT / Markdown / DOCX / EPUB / PDF).

### ☁️ إعداد المزامنة السحابية (التشغيل الذاتي Self-Deploy)

> 💡 **نصيحة:** تطبيق سطح المكتب (نظام التشغيل Windows/macOS) **يحتوي على خادم مزامنة سحابية مدمج رسمي**، ولا يتطلب أي تكوين إضافي لاستخدام المزامنة عبر الأجهزة. إذا كان تكوين Firebase شاقاً للغاية بالنسبة لك، **فمن المستحسن بشدة تنزيل واستخدام التطبيق الثابت مباشرة**.

إذا كنت تصر على النشر الذاتي للبرنامج عبر رمز المصدر (Source Code) أو Vercel وترغب في تمكين المزامنة عبر أجهزة متعددة، فاتبع هذه الخطوات لتكوين قاعدة بيانات Firebase الخاصة بك:

#### 1. إنشاء مشروع عبر منصة Firebase

1. توجه لـ [Firebase Console](https://console.firebase.google.com/) واختر **Create Project**.
2. فُعل التوثيق **Authentication** لكي يتم تسجيل الدخول من خلال طريقة الدخول ← **Google**.
3. قم بإنشاء قاعدة بيانات **Firestore Database** وحدد وضع الإنتاج (Start in production mode) كخيار.
4. قم بتعيين قوانين الأمان (Firestore Security Rules) لتقييد وصول كل مستخدم وحمايته:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

#### 2. تعديل متغيرات البيئة (Environment Variables)

انسخ `.env.example` إلى `.env.local` واملأ قسم Firebase:

```bash
NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id
```

> تستطيع الحصول على هذه الموارد والبيانات بقائمة المشروع داخل Firebase ← الإعدادات العامة ← تطبيقاتك ← إعداد المواصفات (SDK Config).

#### 3. إعداد النشر الخاص بـ Vercel

أضف هذه المتغيرات نفسها إلى **لوحة معلومات Vercel ← إعدادات المشروع ← متغيرات البيئة (Environment Variables)**، ثم قم بإعادة النشر (Redeploy).

> 💡 تم تصميم مفاتيح الـ API الخاصة إلى Firebase لتكون عامة (معرفات من جانب العميل). يتم فرض أمان البيانات من قبل منصة توثيق مصادقة Firebase Auth + وقواعد أمان متجر قاعدة البيانات Firestore Security Rules، وليس عن طريق حجب أو إخفاء مفتاح الـ API نفسه.

## 🔄 دليل التحديث (Updating)

### مستخدمو تطبيق ويندوز لسطح المكتب (Desktop Client Users)

قم بتحميل أحدث إصدار لمثبت التحديث من خلال صفحة [الإصدارات (Releases)](https://github.com/YuanShiJiLoong/author/releases/latest) وقم بتنصيبه فوق النسخة الحالية المتواجدة لديك. بياناتك ومسوداتك القديمة محتفظ بها داخلياً في ملف مسار المتصفح (Browser/Electron profile) ولن تتعرض للضياع.

### مستخدمو أصل الكود (Self-Deployed Users)

#### الخيار 1: التحديث التلقائي المدمج (Auto Update In-App)

قم باختيار **نافذة المساعدة (Help Panel) ← حول البرنامج (About) ← البحث عن التحديثات (Check for Updates)** واضغط على (تحديث الآن | Update Now). سيقوم النظام بدفع وجلب التحديث المعلق `git pull → npm install → npm run build` مباشرة.

> ⚠️ **يتوجب عليك إعادة تشغيل الخادم (Restart the server) فوراً بعد إجراء التحديث كي تأخذ المتغيرات مجراها.** سيعرض التطبيق تلميحات لإعادة تفعيل وتشغيل ذلك.

#### الخيار 2: التحديث اليدوي (Manual Update)

```bash
# 1. دفع واستخراج كود النظام الجديد
git pull origin main

# 2. تنصيب واسترداد العوامل المعتمدة (Dependencies) في حال إضافتها
npm install
# أو بالبديل: pnpm install && pnpm approve-builds

# 3. إعادة بناء البرنامج (مطلوب لبيئة وعمل الخواص الفعلية)
npm run build

# 4. إعادة تشغيل الخادم (Server Reboot)
# في بيئة التطوير محليًا: اضغط Ctrl+C للإيقاف المؤقت، ثم أعد التشغيل
npm run dev

# في بيئة الإنتاج الفعلي: اضغط Ctrl+C للإيقاف المؤقت، ثم أعد التشغيل
npm start

# إذا كنت تستخدم PM2:
pm2 restart author
```

> ⚠️ **تشغيل أمر `git pull` بدون عمل إعادة تشغيل كامل للخادم لـن يطبق مسرى الدعم الجديد أبدًا.** سيبقى عمل خوادم محرك بيئة Node.js يستخرج الكود والمعطيات من الشفرات البرمجية والبيانات المرجعية القديمة المتزامنة مسبقًا.

### مستخدمو منصة Vercel

إذا قمت بتثبيت ونشر النظام بواسطة استنساخ (Fork) أداة Vercel ، فما عليك إلا إعادة مطابقة مستودعك الخاص مع الـ Upstream الأساسي المتواجد في GitHub — وستقوم Vercel تلقائياً بتحديث النشر الخاص بك.

## 🔍 تكوينات وإعدادات البحث عبر الإنترنت (Web Search Configuration)

الأداة Author تدعم تقنيات الذكاء في البحث للحصول واستقطاب المعلومات الفعلية اللحظية. وتختلف المعاملات والطريقة والمسار بناءاً على كل مزود:

| المزود (Provider) | طريقة والية مسار البحث | المتطلبات الإضافية (Setup) |
|----------|--------------|-------------|
| Gemini (النموذج الأصلي) | محرك بحث جوجل الداخلي (Built-in) | لا يتطلب أي تكوينات إضافية |
| OpenAI / OpenAI Responses | بحث داخلي عبر المنصة نفسها | لا يطلب إعدادات إضافية (يتطلب إرفاق اختيار نموذج البحث) |
| DeepSeek / ZhipuAI / SiliconFlow / وغيرها | بحث بمحركات الـ واجهة برمجة تطبيقات (API) الخارجية | **مطلوب إدراج مفاتيح بحث لمحرك مخصص** |

بخصوص مزودي الخدمة الغير مدعومين بنطاق بحث أساسي, يتوجب عليك اختيار محرك بحث خارجي و دمج توثيق الـ API Key الخاص بك فيهم:

### منصة Tavily (مُقترحة وموصى بها — لأنها أسهل للتطبيق)

1. أدخل موقعهم الإلكتروني [tavily.com](https://tavily.com) و قم بخلق وتوليد حساب مستخدم جديد.
2. عقب تسجيل دخولك، اذهب لـ Dashboard واعثر على المعرف الخاص بك والمفتاح (على شاكلة: `tvly-...`)
3. انتقل إلى الإعدادات في حقل الذكاء، ثم (Web Search) واختر نموذج **Tavily** وادخل المفتاح والصقه هناك.

> تمنح الخوادم المجانية: **1,000 إشعار وبحث كحد أعلى للطلب/بالشهر**

### منصة Exa (بحث دلالي Semantic Search)

1. أدخل موقعهم الإلكتروني [exa.ai](https://exa.ai) و قم بتكوين حساب مستخدم مجاني.
2. إحصل على المعرف والمفتاح الخاص بصلاحيات المنصة من [Dashboard](https://dashboard.exa.ai/api-keys).
3. انتقل إلى الإعدادات في حقل الذكاء بالموقع (Author Settings) ثم (Web Search) واختر زر **Exa** وادخل المفتاح.

> تمنح المنصة مجاناً: **1,000 إشعار بحث/بالشهر** — بحث دلالي ومعطيات مُحسنة لاستخدام محركات الذكاء الاصطناعي.

### تخصيص عنوان محرك (Custom Search API URL - Proxy Pool)

أذا قمت بإعداد واختلاق مخزن (Proxy Pool) لمواقع بحث مسبقة الـ Tavily/Exa وذلك باستخدام عدة حسابات ومخازن حرة للاعتماد عليها، يمكنك إدخال **عناوين URL مدمجة مخصصة** لاستخدام تلك العمليات داخل نظام البحث المتضمن للتطبيق:

1. أذهب إلى إعدادات البرنامج في البحث الإنترنت (Web Search) داخل خانة تكوين إعدادات المحرك.
2. اعثر على خيار الإدخال "🔗 عنوان ومسار Custom API URL (اختياري)".
3. ضع مسارك المُجهز في الحقل كطريقة، مثال `https://your-proxy.com`.
4. دعه بشكل افتراضي كما هو وفارغ كي يستخدم الخادم و الوصلات الافتراضية للشركة الأساسية.

> 💡 يقوم النظام بإدراج كلمة وإعداد `/search` بمساره المباشر للعنوان الخاص بك الذي أنشئته — أي أنك لا تحتاج إلى إدراج الكلمة مجدداً بنفسك.

## 🧠 تقنية المتجهات (Embedding) و استرجاع إعدادات العالم عبر حقن الـ RAG

> 💡 هذه الخاصية المتقدمة مصممة ومهندسة لأجل القصص والمجلدات والأعمال السردية المطولة والحبكات الطويلة والروايات التي تحوي في أحشاءها على معلومات عالم ضخمة وملفات الشخصيات هائلة (>20 حقل دخول). أما في حالة القصد بسواها فنجد ان الاستخدام العادي القياسي في القصص الاعتيادية لا يحتاجها على الأطلاق.

### ما هو نظام الـ Vectorization؟ (نطاق تحليل المتجهات)

بشكل افتراضي وتلقائي وعادي جدا، تقوم أنظمة محادثات المولد الاصطناعي الخاص بالخدمة بسحب وحقن **كامل التقارير والإعدادات ومعلومات ملفات الشخصيات** في شريط ومربع الذاكرة الخاص بالمولد. وتبعاً لحالة عندما تتزايد المدخلات والشخصيات وتصبح المعلومات والمخازن هائلة وكبيرة جداً بالكمية، فهذا سيؤدي لضياع المحدودية وتخطي حاجز القياس والسعة لنماذج ونوافذ سياق الذكاء.

لكن عند تشغيل **البحث المُتجه (RAG Vectorized Retrieval)**، فالوضع سيتقلب بالكامل وسيعمل بطريقة مذهلة: سيتم تحويل كافة المعلومات وخصائص العالم إلى أرقام، معادلات ومسارات متجهة فنية حسابية بحتة. وعند بدء الحديث وإنشاء المحادثات، يبحث الذكاء الاصطناعي بسياق آلي وتلقائي بشكل دلالي بحث عن المربعات والمعلومات المُشتقة والمتأثرة بالنص المذكور في المحادثة ويدفع بها و**بالمعلومات الأكثر صلة** فقط في حيز إدخال السرد — بدلاً من رفع جميع التقارير وملأ أشرطة وذاكرة الحفظ ونفاذ الحصيلة المودعة للنماذج.

### متى يستحسن ويتطلب إضافة الميزة؟

| السيناريو والتوقع | توصيات مقترحة |
|----------|---------------|
| حقول دخول للشخصيات وقواعد الإعداد < 20 | لا يتطلب استعمالاً — التحميل الشامل والحقن العام للحيز يعمل على نحو أتم وبصورة مقبولة تماماً |
| حقول دخول للإعداد 20–100 | يستحسن إدراج وتفعيله — لتشخيص ومضاعفة المرجوهات والدقة (recall accuracy) لدى استنتاج وتحليل نماذج لذكاء |
| حقول دخول للإعداد > 100 | إعتماد وتوصيل ذو مستوى حثيث ومرغوب جداً — التفاصيل المهمة للقصة لن تنشق من مدارك الاستعانة ولن تغفل في قاع الذاكرة |

### كيفية تكوينها (How to Configure)

1. من قسم الإعدادات (Settings) استخرج صفحة **تكوين الإتصال (API Configuration)**
2. فعل خيار التبديل والإشعال المتعلق بـ "معابر اتصال API منفردة لحقن التقنية (Separate Embedding API)"
3. قم بملئ الإعدادات التالية وإتمامها:

| المفتاح والمتغير الخاصين | الفكر والوصف المبني والتعقيب |
|---------|-------------|
| **Embedding API Key** | المفتاح الخاص والمعرف الرقمي للنماذج والعمليات المسندة للتبديل (يقبل وتستطيع استعمال ذات المفتاح المحض للشركة ونماذج الدردشة). |
| **Embedding Base URL** | نقطة الإتصال الخاصة بالنموذج API (على سبيل المثال والمشابه لـ: `https://api.openai.com/v1` ومصممة كذلك) |
| **Embedding Model** | اسم النموذج الداخلي: (موصى بضرورة الرجوع لرؤية التوصيات المذكورة أدناه لحسن المبادرة) |

#### النماذج المقترحة والموصى بنفعها (Recommended Models)

| مصمم ومزود الخدمة | اسم ومسمى النموذج المرغوب (Model Name) | الملحوظات والتنويهات |
|----------|-----------|-------|
| OpenAI | `text-embedding-3-small` | تكلفة ونفقة معقولة، وبمحيط 1536 بعد |
| OpenAI | `text-embedding-3-large` | دقة عالية وكفاءة مهيبة تتغلب على النواقص، بمحيط 3072 بعد |
| ZhipuAI | `embedding-3` | نموذج مشدد وتم تخصيصه لثقل اللغة الصينية، بمحيط 2048 بعد |
| SiliconFlow | `BAAI/bge-m3` | طبقات حزم لغوية تدعم تعدد الألسن ومحسنة لها، مع إمكانية استخدام أُطر خدمات دون تسعيرة مفروضة |

### الإجراء وحقن المتجهات بطرق آلية تلقائية (Automatic Vectorization)

- **استشعار واشتعال تلقائي**: فور التوقف والفراغ من كتابتك لتحوير وتحديث أحداث أو بيانات وتعديل على أروقة العالم التابعة لك، يتولد للنظام أسلوب كابح بمدة 3 ثواني، وما إن ينتهي حتى يعالج العمليات ويستحيل إدخالاتك كمتجهات مُحسنة.
- **تحديث وإحلال تدريجي إضافي**: فقط الإدخالات والعناصر المستحدثة والتي قمت بالتعديل على أصولها ومسمياتها سيتم التكافل بإرساء معالمها، وهذا التطور يُعين على الإطاحة والحد من مشاكل الكادر المُتجه ومساعيه المهولة واختصار النفقات والكوتا (Quotas).
- **الإرتكاز المحلّي**: تحفظ وتنزل البيانات المتعلقة بعمليات التوجيه المتجه (Vector data) بأمان بالغ على جزيئيات وحيز متصفحك الأساسية وتحديداً (IndexedDB)، فلا تلتفِت للسرقات والتسريب حيث لن تقع بيانات المتجهات والعمليات التوجيهية في يد مخدم الويب للشركة مطلقاً.
- **استهلالية المبادرة وتشغيل التلقين الآلي**: في لحظة وضع وربط وحقن المدخلات والمشروعات عبر الإعدادات عن طريق استيرادها او تحميلها بصور السحابة المركزية، سوف يقوم البرنامج في الخفاء وصورة غير مشاهدة بإنزال المجلدات المعالجة وتشغيل آليات الإشتقاق بشكل مدهش دون حوافز.

### إطلاق وإحداثية إعمار التجديد الخالص بالبناء المرفق يدوياً (Manual Rebuild)

لو أحببت وأقدمت على إجراء وتحويل وتبديل النماذج المعنية والمتهمة بموجبات الحقن وتقنية المتجهات او قد واجهتك مشكلة نسيجة بسلامة الإحصاءات الرقمية للمعالجات، تستطيع إعادة إخراج وتكوينات الأسلوب الآلي يدويا:

1. افتح صفحة (Settings) وألج لخانة **API Configuration**
2. اضغط على خيار المعالج وزر "**تحفيز بناء المعرفات المؤشرية المتجهية مجدداً | Rebuild Vector Index**"
3. أصبر وتوارى عن فعل أمر حتى يقوم جميع معالجاتك ومواثيقك المدمجة للإدخال بإستخلاص ووضع العمليات من أصولها بشكل جديد من البداية

### إجراء وآلية تدفق سير العمليات بالنسق الداخلي (Workflow)

```
يدخل المعني محرر تعديل إدخالات وإعدادات البيانات ← ينتظر 3 ثواني لمقاومة الاهتزاز الميكانيكي ← يتصل بنداءات الـ API المختصة بجلب قيم المتجهات لترسيخ النموذج
                                                    ↓
                                           يرفق بالحيز للتخزين (IndexedDB)
                                                    ↓
مفوضات محادثات الـ AI والذواكر ← حقن المدخل والنص الوارد من الإنسان بمفهومات المجهات ← خوارزميات تتطابق بفعالية للحقن المُشتق الجيبي (Cosine similarity matching) ← ترفع وتصرف التشكيلات الـ Top-K للمساقات في الذاكرة لتندمج مع الوحدة!
```

## 💾 وثائق أسلوب وهيئات التنسيق لآليات استيراد وتكوينات العوالم

المحرك Author يرسخ بدعامة مبجلة ويقبل تحميل ووضع المدخلات من أصول وتنسيقات مختلفة: **JSON / Markdown / TXT / DOCX / PDF**.

### القوام الرئيسي المركزي: المستويات والنظم الهرمية الأربعة (Four Tiers)

تستعمل قوالب وموازين الاستيراد المُتبعة **رموز ومواثيق وشعارات هيكلية دقيقة (structural markers)** لأجل المضاهاة وحفظ الصلاحيات التفريقية للجزئيات والسمات الأربع المقسّمة أدناه. بالرغم من اختلاف واستحداث أمتزجة أنواع صيغ وصيغ الملفات المستوردة المرفقة في هذه الخوذة، سيكون محتماً وألزاماً الإبقاء وتخويل واحتواء التسلسل البيني للدمج التالي:
`تصنيفات (Category) ← مُدخل/عنصر (Entry) ← علامة/حقل (Field) ← نواتج المُعطى (Value) (قد يحتمل سطراً مركبا يتشكل من سياقات طويلة تتوالى)`

وبهذا، **تستطاع أي محددات وأي قوام وخصائص ونظم دلالية معدلة ومبنية لأجلك أنت أيا كان شكلها أن يتم استلامها واستخلاص ماهيتها لتُحفظ بكافة البيانات بلا إضمار أو ضرر وبدقة واسترداد بلا نواقص**.

### موثوقية تنسيقات المعطيات وترتيبات مراجع الأهمية (Format Reliability)

| مستوى الصدقية | أساليب ومعطيات التوثيق | الفهرسة والتصنيفات (Category) | قوام الُمُدخل (Entry) | الحقل المقيس (Field) | خطوط المُدخل البينية المتراصة (Multiline) |
|------|--------|----------|-------|-------|-----------|
| ⭐⭐⭐⭐⭐ (قيمة مصونة وثابتة) | **JSON** | كـ مفتاح `category` | باسم `name` | مستمدة بمفتاح الـ `content` بنهج (key-val) | علامات دفق الخانات المُشار إليها كـ `\n` |
| ⭐⭐⭐⭐⭐ (أصيلة بلا نقص) | **MD** | بنقطتات وتوجيهات `## Category Name` | دلالات `### Entry Name` | مسماة بنهج `**Tag**: Value` | الفراغ البادئ بسياق المسودة (Indented text) |
| ⭐⭐⭐⭐ (نطاقات مأمونة ومحفوظة) | **TXT** | بإيحاء وحيز `│ Category Name` | مُرتبطة ودالة بـ `【Entry Name】` | مُعلّمة وواقعة في `〈Tag〉: Value` | التصدير بمسافة مقاسة وفراغات (Indented text) |
| ⭐⭐⭐ (محفوظة بتماثل مألوف وتقارب) | **DOCX** | بتوجه نصوص ميزة الـ (Heading 1) | بنمط وميزة الـ (Heading 2) | علامات الـ **عريض الدكنة (Bold)** وبحيز نقطتين (colon) | وضعية المفقرة وتمركز أثر النص المُستطرق |
| ⭐⭐ (درجة حفظ ومعايرة عالية للوصف البسيط) | **PDF** | شارة وصندوق `■ Category Name` | معين ونجمة `◆ Entry Name` | سهمي `▸ Tag: Value` | فضاء المسافات والتدرجات للمسودات |

### الهيكل القياسي وأمثلة لصنع مُسودات وتفصيلات הـ Markdown (المفضل والموصى باتباعه)

إلزم ديباجة وشارح `##` لإبراز التخصيص والمجموعات، واحتكم لـ `###` كشارة بارزة لبناء ومعرفة اسم المكون والعنصر الخاضع للشرح، واقتبس ووضب `**Tag**: Content` كتخصيص ومعلم للسمات:

```markdown
## أي من مجاميع التصنيف وأوجه التخصصية القابلة للإبداع هنا

### السيد فلان (أو اسم المكون)

**الوضعية والدور بالحبكة**: شخصية البطل الحكوي
**خلفيات ماضي المتخفي المبهمة**: وُلد وترعرع بعمق جبال السنديان الشامخة.
  ولطالما واظب بتشوق باهر على التلمذة على أساسيات الجري وتصاريف الأسلحة برفقة معلمه.
  ولم ينفك المُعلم مردداً: "لن يستطاع أحد سبر وجلب تلك الكشوفات إلا بيُمناك."
**فصل ودلالات الأنساب السحرية الفلكلورية**: عرق يُخالج سُبات طوائف التنانين المبجلة

### السيدة زرقاء
**الدنس والجنسية**: أنثوي
**حواضر الطابع النفسي المتبين**: متزنة الملامح وتلتحف برونق البهجة، ولا يستهان بقوة بصائرها ورصيد التتبع للمهملات
```

### صياغة وأمثلة لنُظم الـ TXT القياسية الثابتة

نستحضر معطيات الاسترجاع باستعراض إزاحة الأشرطة `│` للدلالة على الصندوق المستوعب للمجاميع، ولواحق الإمتياز الكرتية `【】` لأسماء الكيانات والعناصر المدرجة، وصياغة `〈Tag〉:` لتعيين مسارات المُعلقات الدلالية الملحوظة:

```
┌──────────────────────────
│ أي من المخرجات وتصنيفات الجموع الحاصلة
└──────────────────────────

【فلان الفلاني】
〈الجذور والنطاق الجندري〉: شخصية ذكر ذكوري
〈معلومات وقصصية السلف المنثورة〉: نشأ في ظلال السهول الخفية على أذهان العالمين.
  بادر طوعا وتأهباً بتدارك وتلقين شتى مجالات الفنون مع أستاذه المتطلب والمتجبر.
  وسرعان ما ألقى عليه جملة شتت وعي عقله: "أين هي؟ أجد المسألة بحوزتك وحدك."
〈السلالات الغريبة التي تركن بدمائه〉: نسل محاذي لثُلة أباطرة وأشراف الدم الأزرق
```

### ترتيب وصياغة محاذات الـ DOCX القياسي (Microsoft Word)

في وسط متون أجهزة النوافذ لبرامج Word، استحوذ وتفحص أنماط ميزات الخطوط **Heading styles والتغميق العريض والمكتنز (bolding)** لمفاضلة مبيانات المُعطاة الأساسية للمقروء:
- **تحجيم النمط التوضيحي رقم واحد (Heading 1 أو H1)** → كدلالة على قسم التصنيف واسم المجموعة (على غرار ووزن، "شخصيات الإحتدام والمواجهة المركزية المرجوة")
- **تحجيم النمط التوضيحي رقم الثان (Heading 2 أو H2)** → يوضع حافزاً ومستخلصاً لمسبار مسميات العنصر (ومثل ذلك، "فلان من الناس")
- **بداية الاسترسال الأول في متن المقال** → يجب أن تُغمّق الحواضر والمسميات بـ **الخط العريض والمُسمن للحقول الداعية (Bold Field Name)** ومرتزق ومتبوع بنقاط الشرح الأساسية: المحتوى المدخل الشارح (ومثلما سلف، **سلالات ونخب فرعية**: طوائف وشتى من أباطرة التنانين الزرقاء المُرعبة)
- **إتباع مُلتحق للمتن وتسطيرات الشرح المحشو** → ستُضاف المسارات وسياقات المسودات المدخلة المُصاغة المتراصة (Multiline) بمسافات إبتدائية مُستلزمة للإيضاح الدقيق للحقيقة ككل (first-line indent).

### 💡 آفاق واستلهمات التصميم المُتوافق وملاحظات التنبيه المتوقعة (Compatibility Design & Notes)

1. **تقنية واحتراز العودة للتعيين ومعايرة السمة والإسقاط (Auto Tag Fallback - Backwards Compatibility)** — عندما يغدوا هناك انحسار وتسقط دلالة هيكلية أو مُسطرة ضمن بيانات الـ TXT أو أروقة الـ Markdown (مثل توظيف نص `الاسم المسمى: فلان الفلاني` بدلا واستغناءً عن توظيف المقابح الصحيحة `**الاسم المسمى**: فلان الفلاني`)، سيحاول النظام بخاصية ذكية ومعالجة إدارية أن يقوم بالالتزام والتماشي مع الأمر واستخراج وفهم السياقات المعناة برحابة صلح *ولكن يقتصر ذلك على شرط حتمي وحيد* وهو أن تعود وتؤول وتصنف المعاني المرادة والمنقوصة كالـ `الاسم المسمى` لزمرة وضمن النطاقات المتنبأ بها كجدران وحقول قياسية أساسية متعارفة بالبرنامج والأطقم الأصلية (core field set). ومما استلزمنا لقوله وإيقاظه: إن كنتم تبغون استنزال دقة لا محال عنها وتساوي نسبة 100% لاستخلاص وتعيين دقة مُفرزات الجُمل والمقاطع الدالة على المخصصات الجُزئية من قِبلكم (custom fields)، فيتعين ويتوجب وينبغي ضرورة إلزامية حتمية موازاة ومساواة هيكلة وصيغة المواثيق والتنسيقات البارزة التشكيل المعترف بها والتحلي بها.
2. **مجال مُسایرة وحَسن تناول قيم المُستخرجات والإفادات الداعمة للأسطر العريضة الُمتعددة (Multiline Value Handling)** — في كُل وأي حين وزمان تستفتح مُلزماتك بوضع دلالات `**وصف وتقييد التحديد (Tag)**: ` أو إشارات `〈إدراج دلالي للسمة (Tag)〉: `، فإن ما سيتلقاه وما سيأتي بعد هذا وسيلحقه من أسطر دليلة ومستقطبة **تتطلب مسافة فراغ تعادل وتوازي ضربتين من المسطرة (2 spaces indent)** ليتم التوفيق وحَملها على أنها البدن والمحتوى والجزء الُمنشق للحقل ذو السعة (كمضامين إستخلاصات للقيم العظيمة مُتعددة الأسطر)، وسيظل هكذا يتلو بعضه ويسرد تباعاً حتى تتصارع وتشاهد وتلاقي مسارات الاستخلاص نقطة دلالية ذات سمة وشكل قاطع و هيكلي من نمط ومضمون آخر أو تتصل بالمحطات.
3. **قصور وعقبات وآفات ومحدوديات تصنيف الإنزال والاستدلال في صيغ DOCX / PDF واستنباطهم (Parsing Limitations)** — تُهيكل ملفات وصيغ حِزم الـ PDF وتُجزأ عبر الإفصاح عن شارة ومسلك إستكشاف مُصمم ومُتخذ بصورة مقطعية مُستحدثة للنداء كـ `▸` ومُعطياتها. إن كَان ما تم الاستجلاب والمجيء به واستنباطه وملحق رفعه كملف بصيغة PDF/DOCX ولم يكن بالأساس وليد أو مخرجاً وحصيلة أو ناتج صُنع لمُركب البرمجية ومُصنعتها الأصلية (أي مخرجاتها من داخلها)، سيحاول ويرتأي وينهل أحد المحللين والفاصلين المتمرسين في المنظور الإدراكي والتخيل والتخمين (Heuristic parser) المُكلفين بصدق أن يستخلص الدوامات والمُنتشرات والمعطيات من جوف النص على أقصى وأفضل الأوجه المُمكنة الُمتاحة، غير أن الضمان بصلاحية واستخلاص الدلائل والمُخرجات بشمولية نسبة تطابق ورسم محاكي يعادل 100% (High fidelity) سيكون محل تشكيك ومجازاة محال التكهن والوثوق المطلق بها حتمياً بلا نكسة.
4. **شكل وتنسيق ومرجعية الـ JSON تمثل وتستحوذ الركيزة المتناهية والحصرية للكمال والاستيعاب (The Most Complete)** — وبدون استنكار ومعارضة وبلا جدلية، تُثبت موازين وحويصلة مقاييس وأحكام صيغة الـ JSON أنها التنسيق المقطوع والمُشرع الأكثر شدة وصرامة ودقة على الإطلاق، وهو ما يجعله الوحيد وبصيغته المتفردة المعوّل عليه والمشخول والممتهن لإتاحة القدرة ونقل وتغليف ومهاجرة تمامیة وتطابق إحصائيات وخصائص وإعدادات كُل وبشكل قاطع كافة الـسمات المدمجة والمعنوية للمشاريع (بما يتصدر ويشتمل على أطوار وضعية وحالة وأنماط الإنشاء (writing mode) والكتالوج الاستعلامي والمعلوماتي والمشروخ للمشروع برمته project info). بناءاً على هذا، وإذا كان مقاس الأمر ومُنحناه ومدارك الاهتمامات هي لا تزيد عن عملية تبادل وتحويل وسائط وأجهزة الإنشاء والآلات فقط الباحثة عن تنزيل الموارد والانسيابية، فإنه يُنصح وينبثق الوعاز الُمحكم المرجو بضرورة وإلزامية الإتجاه واستخدام واستهلاك سُبل استيراد أو تصدير المشاريع بكامل أحوالها بهيئة وضِيافة واستحواذ مُتضمن لصيغة ونَسق `JSON` كمنشط ومنهاج رئيسي ومستديم، بلا خلل ولا غبن مُحذر أو تردد.

## 🔒 الخصوصية وأمن المعلومات التام (Privacy & Data Security)

### التخزين والفهرسة والاستقرار الُمحلي المركزي (مأمونة وواقعية وآمنة تماماً)
- كُل بيانات وصناعة الأفكار والجمل والإبداعات الوضاءة التي تصيغها حواسك (المسودات والفصول، التهيئات وخصائص النظام الروائية، وحالات وأنماط الـ الحفظ الـمأخوذة والمقطوعة Snapshots) **موجودة ومخزنة ومستضافة بسلام وبنسبة تحصي واستيثاق 100% بصورة ونطاق محلي تماماً يقبع داخل مجلد وقواعد وأُطر المتصفح الخاص بآلتك وبريادتك (المعروف بـ IndexedDB)** — دون أن تُرسل، تُدفع، أو تُرفع أبداً ولم يُجأر بها نحو أي خوادم أو مزود سحابي منسي أو مركون بتاتاً!
- يتم اصطياد ومعالجة وتخزين مُفاتيح وأسرار وإشارات التواصل للـ (API Keys) ضمن إطار مُؤمن ومرتع للذاكرة المصلتة للمستعرض وهو ما يسمى بـ حيز التخزين المحلي للمتصفح (browser localStorage).

### ⚠️ مجريات وانسياب وسراديب مسار بياناتك وإحداثي المُعطى عند إقدامك ومُلاقاتك لإستخدام قدرات وعمليات وتفضيلات المُولد والذكاء الاصطناعي (Data Flow)

عند تعقيد واتساق وتشغيل ودخولك في معترك ومضمار قدرات وحقول الذكاء الاصطناعي واستباحة تفاعلها (المتمثلة في وضع استكمال السرد، تغيير وإعادة هندسة الجُمل المتواجدة، المحادثة والتشات الحُر، إلخ المتفق)، فأن الحصيلة والمحتوى الدلالي والأرقام التالية للمرسل تمر وتجتاز وتعبر وتصل إلى نقطة عبور واصطفاف متعلقة بـ **الخادم الموثق المُستضيف للناشر (deployer's server)** في مسار سيره وتوجهه وانطلاقه ناحية مُصنع ومزود الخدمة الاصطناعية (AI provider) لاسترجاع المردود:
- صمام والمفتاح المتداول ومُفتاح العبور السري الخاص فيك كبناء للـ **أداة والرقم المعرفي (API Key)**
- **حروفك وبدن والوعاء النصي وسياقات المكتوب المحض (text content)** وهو المكون والمطلب المادي الأساسي الذي تفضت وألقيت وبثثته للذكاء.

```
واجهة جهازك المتصفحي ← خادم وجدار الوحدة الخاصة للناشر الذي نصب النسخة ← شركة ومزود الذكاء الآلي (ZhipuAI/Gemini/DeepSeek/etc.)
```

**وفي حالة كنت لا تلجأ ولا تستغيث إلا بمنافذ ومستخدم خفي وتستقي من مرتع مستنسخ وسيرفر عام مفتوح قام بنشره وجهود ملموسة لطرف ثان وللعموم (public instance)**، فعلى الرغم من أن المتصدي وصاحب المنصة والمُدير يقن ولا يؤيد المراقبات أو يعد ويوثق بكفالة عدم الاستراق وتحليل الـ logs (السجلات وذاكرة المشاهدة)، فأن المُكنة والطاقة والاستطاعة الجبرية والاختراق الفني (technical capability) لترصد البيانات وشن عمليات الإيقاع للمساق واعتراض دفق البيانات (intercept) واردة ومتاحة حُكماً في الساحة وقائمة بحد ذاتها كشبح غير مكفول. وبناءً عليه وبحسبه:

1. ✅ لك الحُرية في الانخراط والإعتماد الموقوت واستعمال وسحب صلاحيات وحُلة المنصة المفتوحة للناس أجمع بغرض **أخذ غمار التجربة المُستعجلة السريعة والأختبار الخفيف (quick trial)**
2. ⚠️ وما إن تفرغ وتكتفي من لمس وتجربة البرنامج وتذوق قدراته وثناياه، **فهرول وابدأ وانطلق بلا توانی وثبط بتدمير ومسح وصعق والتبديد المطلق لكل مفتاح API استعملته والمفتاح الخاص لك بلا إرجاع عبر الولوج لموقع الشركة ومزود خدمتك الرسمي (destroy your API Key)**
3. 🔐 **وبلا محيد أو تراجع وفي حال كانت النوايا مُترسخة للعمل والإبداع الواقعي والمُستمر بحماية لا شائبة فيها، قم وتفضل بسل واستنساخ الـ (fork) وخصص ونصب (deploy) كينونة الاستضافات المرجعية ومرجعيتك الخاصة و الخادم الحاضن والانفرادي بك — لتصبح وتستحيل وتؤول كل انسكابات وقوافل مسار البيانات كلياً متقوقعة وبأمان ولن تعرج ولن تُسير سوى في عروق وممرات ومخارج سيرفرك وخادمك المبرمج الذي تعتمده أنت وتملكه وحدك سرياً (own private instance)**

> 💡 لتركيب ونصب وتجهيز وخلق مثيل و نسخة محكية مُتوافقة وقاعدة مدمجة لك أمر لا تكبد ولا طاقة تُذكر به وسهل المنال: خذ نسخة (Fork) من هذا الـ Repo ← ضغطة ومُعانقة واحدة لدفع ولنصب واستزراع الكود في منصة (deploy to Vercel) ← وتمت الإنهائة والنجاح. العملية بحذافيرها والخطوات تستغرق أقل من 5 دقائق وتكتمل.
